### Список основных git комманд

---

```
git init
```
- Инициализирует репозиторий в текущей папке

---

```
git add <filename>
```
- Добавляет в коммит текущие изменения в указанном файле.
- Если файл был изменен после его добавления в коммит и до создания коммита, эти изменения не попадут в коммит. Чтобы последние изменения попали в коммит, нужно выполнить git add <filename> еще раз.
- Если файл находится в подпапке - нужно указывать путь к нему относительно текущей папки. Например: `git add dir1/dir2/file.txt`
- В комманду можно передать несколько файлов сразу. Все они будут добавлены.
- Можно добавить все существующие в текущей ветке изменения: `git add .`

---

```
git status
```
- Выводит текущий статус локального репозитория в текущей ветке.

---

```
git commit -m "commit message"
```
- Создает коммит из текущих отслеживаемых изменений, добавленных командой `git add`.
- Если нужно добавить в коммит все изменения, можно не выполнять `git add` перед созданием коммита. Вместо этого можно выполнить `git commit -am "commit message"`. Ключ `-a` (all) добавляет все текущие изменения в коммит.
- Коммит можно создать без сообщения. Например, `git commit` и `git commit -a` - валидные комманды.

---

```
git push
```
- Отправляет (пушит) все коммиты, существующие в текущей ветке, в удаленный репозиторий (хранилище), делая код доступным для других пользователей.

---

### Другие основные комманды

---

```shell script
git remote add <remotename> git@github.com:username/reponame.git
```
- Указывает адрес удаленного репозитория, с которым будет связан текущий локальный репозиторий.

```
git clone git@github.com:username/reponame.git
```
- Клонирует указанный репозиторий. В текущей папке будут создана папка с названием репозитория, содержащая копию удаленного клонируемого репозитория.
- Склонированный репозиторий содержит абсолютно все ветки, информацию о коммитах и т.д., что и удаленный репозиторий.

```
git log
```
- Выводит лог коммитов в хронологическом порядке, от самого последнего (вверху списка) к самому первому (внизу списка). Передвижение по списку осуществляется стрелками вверх вниз. Для выхода нужно нажать `q`.

```
git reset HEAD~
```
- Удаление последнего созданного коммита. Все изменения при этом остаются, т.е. удаляется только слепок (снэпшот) изменений.

#### Ветвление (бранчевание)

---

```
git branch
```
- Выводит список всех веток, существующих в локальном репозитории.
- Можно так же вывести список всех веток в локальном и удаленном репозиториях: `git branch -a`

```
git branch <branchname>
```
- Создает ветку с указанным именем.
- Ветка не будет создана, если в текущем репозитории нет ни одного коммита.

```
git checkout <branchname>
```
- Переключает текущую ветку на указанную.
- С помощью ключа `-b` можно совместить создание новой ветки и переключение (чекаут) в нее: `git checkout -b <branchname>`

---

```
git merge <branchname>
```
- Слияет (мержит) указанную ветку в текушую. Например, если нужно смержить ветку `foobar` в `master`, нужно сначала переключиться в `master`, затем смержить ветку `foobar` в текущую ветку (т.е. в `master`):
```
# переключение в ветку, в которую будет сливаться целевая ветка
git checkout master
# слияние целевой ветки (foobar) в текущую (master)
git merge foobar
```
ВАЖНО!
Любые действия, выполняемые в локальном репозитории, не появятся в удаленном репозитории до тех пор, пока не будет выполнен `git push`.  Поэтому после слияния двух веток нужно выполнить `git push`. После этого изменения, произошедшие в рез-те слияния двух веток, будут доступны другим пользователям.

```
git pull
```
- Вытягивание (пулл) изменений из удаленного репозитория в текущей ветке в локальный репозиторий.

#### Git Workflow

---

Стандартный гит воркфлоу. Следующий порядок команд подразумевает следующее:
- удаленный репозиторий существует, и он не пуст: у него обозначена `master` ветка и он содержит в себе как минимум 1 коммит
- локальной копии удаленного репозитория нет

```shell script
git clone git@github.com:username/reponame.git
git branch foobar
git checkout foobar
# make changes
git add filename
git commit -m "made some changes"
git push -u origin foobar
git checkout master
git merge foobar
git push -u origin master
```
